# VMIR2 Lowering

## The Golden Rule

The Golden Rule: Everything is a normal function call.

Add two integers? That's a call to `OPERATOR+`.
Dereference a pointer? That's a call to `OPERATOR->` or similar.
Convert a value to another type? That's a call to `.CONSTRUCTOR`.

Do not under any circumstances introduce new code that breaks the golden rule without explicit human approval. Flag existing code that breaks the golden rule for review as you encounter it.

## Built-in functions

Built-in functions can be divided into two groups: *Intrinsic Builtins* and *Generated Builtins*.

In _both cases_, a builtin function emits an overload during overload resolution. No short circuits! Do not bypass this, EVER.

functum_builtins MUST return an overload for the called functum. Always. No exceptions, no short circuits, no branches that bypass this routine.

Emission of VMIR2 _instructions_ happens through so-called intrinsic lowering. Not randomly placed if statements throughout the code, not via randomly placed conditions and special cases. emit should almost never be called anywhere else. Agents REQUIRE human approval before adding a call to `emit`, avoid it if you can.

Intrinsic-Builtins produce instructions using intrinsic_instruction. This produces a _single_ instruction. When an intrinsic routine is "called", no actual invoke instruction is emitted, instead an instruction is generated. 99% of non-terminating instructions should be generated this way.

The second pathway for a builtin routine is Generated-Builtins. Generated routines are for complex non-primitive operations, like builtin constructors, that may need to do multiple things. Generated-Builtin routines work the same as regular routines, except that instead of their code being generated based on user supplied function bodies, it's generated by the compiler.

## Intrinsic-Builtins

`std::optional< vmir2::vm_instruction > intrinsic_instruction(type_symbol func, codegen_invocation_args args)`

This is the sole, and only, entry point that determines if a builtin function call is an intrinsic. Nearly any case you need to add
an instruction goes inside this function. Exceptions require escalation and manual approval. Simply return the instruction, or nullopt if the builtin call is not an intrinsic instruction.

## Generated-Builtins

When a function call is a builtin, it first checks if the builtin is an intrinsic, otherwise it falls through to generated routines.

Generated routines are for complex multi-instruction cases.
